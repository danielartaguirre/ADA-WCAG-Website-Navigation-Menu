ADA & WCAG Compliance

The Americans with Disabilities Act (ADA) and related laws treat WCAG 2.0 Level AA as a common legal baseline. Many organizations now adopt WCAG 2.1 or 2.2 Level AA as best practice to provide inclusive experiences and reduce legal risk.

  • Americans with Disabilities Act (ADA)
  • Section 508 of the Rehabilitation Act

WCAG references:

  • WCAG 2.0
  • WCAG 2.1
  • WCAG 2.2
  • WCAG 2.2 Quick Reference

WCAG 2.0 compliance

Our navigation and footer follow key WCAG 2.0 Level AA guidelines for consistent structure and accessible links.

  • Skip link addresses 2.4.1 Bypass Blocks
    W3C · WCAG.com
  • Consistent footer links meet 3.2.3 Consistent Navigation
    W3C · WCAG.com
  • Link labeling supports 3.2.4 Consistent Identification
    W3C · WCAG.com

WCAG 2.1 coverage reference

Structure is conveyed programmatically via landmarks, lists, and headings so AT can interpret relationships.
W3C · WCAG.com

All functionality is operable through a keyboard without specific timing. Our nav, toggler, and dropdowns are keyboard-usable.
W3C · WCAG.com

Keyboard focus is not trapped in any component; users can move focus away using standard keys (Tab/Shift+Tab or Esc).
W3C · WCAG.com

Provide a way to skip repeated content. We include a visible “Skip to main content” link that targets #main-content.
W3C · WCAG.com

Focus moves in a meaningful sequence that preserves relationships. DOM order: skip tray → header/nav → main → footer.
W3C · WCAG.com

Components expose their name, role, and state to assistive tech. Our dropdown trigger is a button with aria-expanded and aria-controls.
W3C · WCAG.com

Text and interactive elements have a contrast ratio of at least 4.5:1 against adjacent colors to support users with low vision.
W3C · WCAG.com

Content reflows without loss of information, and custom text spacing does not break layout. Bootstrap’s responsive grid helps with these.
W3C Reflow · W3C Text Spacing · WCAG.com Reflow · WCAG.com Text Spacing

Headings and labels describe topic or purpose. We use semantic headings and clear link text.
W3C · WCAG.com

A visible focus indicator is present for keyboard focus. We add a high contrast outline with offset.
W3C · WCAG.com

WCAG 2.2 coverage reference

When help mechanisms are present, they appear in a consistent location across pages. The footer contains stable contact/help links.
W3C · WCAG.com

Keyboard focus has a clearly visible indicator with sufficient contrast and area. We apply a 3px outline with offset to avoid being obscured.
W3C · WCAG.com

The focused element is not fully hidden by author-created UI. We use outline offsets and scroll-margin on the main skip target.
W3C · WCAG.com

Pointer targets are at least 24 by 24 CSS pixels or have equivalent spacing, with listed exceptions. Our interactive elements meet or exceed 24×24; 44×44 is used where comfortable.
W3C · WCAG.com

Motion triggered by interaction can be disabled unless essential. We respect prefers-reduced-motion and suppress transitions/animations accordingly.
W3C · WCAG.com

The focused element is not hidden at all by author-created UI. We use outline offsets and scroll-margin on the main skip target.
W3C · WCAG.com
